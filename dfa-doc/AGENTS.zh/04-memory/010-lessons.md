# 经验教训与知识记忆

## 术语表与知识记忆 层契约

- 每条 lesson 都应绑定具体失败或回归场景，作为可复用规则而非叙述。
- 用可执行的祈使句写 lesson，确保后续 agent 可直接照做。
- lesson 过期时，追加“日期 + 触发条件 + 新规则”覆盖说明，不删除旧上下文。

## 持久化经验记录

- 在大规模结构修改前，请务必阅读入口文档 (entry) 和架构文档 (architecture)。
- 在发生明显的仓库形态变更后，请立即刷新生成的智能体文档。
- 优先使用明确的契约和稳定的名称，而不是让智能体自行发挥命名。
- 保持清单文件、README 示例与生成器行为一致，确保技能不出现“过度承诺”。
- 混合的仓库信号是一种警告，在跨边界重构前请务必仔细检查。

## 经验日志（仅追加）

- 按时间追加人工确认的里程碑、复盘结论和术语决策。
- 历史记录只追加修正，不做整段重写。
- refresh-safe 规则：本节内容在 refresh 时默认保留；若某段需要更强保护，可再使用 manual block。
